* geiser-kawa (Momo Softworks fork)

This repository is a **fork** of the original ~geiser-kawa~ project hosted at [[https://gitlab.com/emacs-geiser/kawa][gitlab.com/emacs-geiser/kawa]].
It is maintained by **Momo Softworks** and adds a few conveniences for the
KawaCraft server setup while preserving the original functionality.

**Original author**: The upstream code was written by the original maintainers of the
~geiser-kawa~ project.  All credit goes to them; this fork only adds packaging and
documentation updates.

**What it is**
- An Emacs [[https://gitlab.com/emacs-geiser/geiser][Geiser]] backend that provides Scheme evaluation, loading, completions, and
  autodoc for Kawa 3.1.1.
- Includes a small Maven wrapper that builds the Java side (the ~kawa-geiser~
  JAR) and ships the required Kawa 3.1.1 runtime.

**Supported versions**
- Kawa 3.1.1 (the version bundled with the generated JAR)
- Geiser 0.32 (required Emacs package version)

* Installation

1. **Prerequisites**
   - =java= (JDK 8+), =emacs=, and =git= on your $PATH.
   - =JAVA_HOME= environment variable set.
2. **Clone the fork**
   ```sh
   git clone https://github.com/Momo-Softworks/geiser-kawa.git
   cd geiser-kawa
   ```
3. **Build the Java dependency JAR**
   ```sh
   ./mvnw package
   ```
   This creates ~target/kawa-geiser-*.jar~ containing Kawa 3.1.1 and its
   dependencies.
4. **Add the Emacs source directory to your load‑path**
   ```elisp
   (add-to-list 'load-path "<absolute‑path>/geiser-kawa/elisp")
   (require 'geiser-kawa)
   ```
   Or, if you use ~use-package~:
   ```elisp
   (use-package geiser-kawa
     :load-path "<absolute‑path>/geiser-kawa/elisp"
     :after geiser)
   ```
5. **Start a Kawa REPL** (e.g., with ~M-x run-kawa~) and enjoy Geiser support.

* Using with KawaCraft

KawaCraft runs a Kawa REPL server on port 4242 (client 4243).  The fork provides
two helpers to make connecting Emacs to that server easier.

**geiser-kawa-extra-classpath**
- Variable: ~geiser-kawa-extra-classpath~
- Purpose: Adds additional JARs or directories to the classpath of the
  Kawa process started by Geiser.  Useful for loading KawaCraft‑specific
  libraries.
- Value: a *list of strings* (JAR files or directories).
- Example in ~.dir‑locals.el~:
  ```elisp
  ((scheme-mode . ((geiser-kawa-extra-classpath
                    . ("build/rfg/recompiled_minecraft-1.7.10.jar"
                       "build/classes")))))
  ```

**geiser-kawa-connect**
- Function: ~geiser-kawa-connect~
- Purpose: Connects an existing Geiser REPL to a running Kawa server (e.g., the
  KawaCraft server).  It opens a socket on the given host/port.
- Typical usage:
  ```elisp
  (geiser-kawa-connect "localhost" 4242) ; connect to KawaCraft server
  ```
  After connecting you can evaluate code against the live server.

**geiser-kawa-jump-to-java-definition** (member-precise =M-.= for Java)
- Command: ~geiser-kawa-jump-to-java-definition~ (bound to =M-.= in Kawa
  buffers).  On a Java member at point (e.g. ~(java.lang.String:format ...)~ or
  an instance method) it asks the live REPL for the member's owning class and
  the exact source line of each overload, then opens the source at that line.
  With several overloads it prompts (=completing-read=) for which signature.
- Variable: ~geiser-kawa-source-path~ — a *list of strings*: source archives
  (=-sources.jar= / =src.zip=) or source directories to search.  The JDK
  =src.zip= is added automatically when it can be located from the ~java~ binary.
- How it works: precise line numbers come from the compiled class's
  ~LineNumberTable~ (read with ~javap -l~), which RetroFuturaGradle's recompiled
  Minecraft classes carry and which map exactly into the MCP sources jar.
- Example in ~.dir‑locals.el~ (KawaCraft):
  ```elisp
  ((scheme-mode . ((geiser-kawa-source-path
                    . ("build/rfg/mcp_patched_minecraft-sources.jar"
                       "build/libs/kawacraft-0.1.0-sources.jar")))))
  ```

* Feature List

- **Geiser‑related**: eval, load‑file, completions, macro‑expand, autodoc for
  Scheme procedures and Java methods, manual lookup.
- **Kawa‑specific**: Java classpath handling, `geiser-kawa-capf` completion for
  corfu/company, `geiser-kawa-extra-classpath`, `geiser-kawa-connect`,
  member-precise Java jump-to-definition (`geiser-kawa-jump-to-java-definition`,
  `geiser-kawa-source-path`).
- **KawaCraft integration**: see the section above.
